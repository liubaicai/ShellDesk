# 终端组件重设计文档

## 定位

终端是 ShellDesk 远程桌面的基础能力，不应只是一个 SSH 输出框，而应成为远程工作流的命令执行核心。重新设计时优先考虑长时间使用、并行排障、可恢复上下文和与其他组件协作。

## 重新设计目标

- 让单个远程桌面可以承载多个终端会话，而不是把所有工作压进一个 shell。
- 让执行长命令、查找历史输出、复制排障片段更顺手。
- 让终端能和文件管理器、命令收藏夹、部署面板、会话录制共享上下文。
- 保持终端本体轻、快、可靠，避免把它做成半个 IDE。

## 功能架构

### 会话层

- 多标签终端，每个标签独立 title、工作状态和关闭确认。
- 支持拆分视图，至少支持左右或上下两分屏。
- 新建会话可选择默认 shell、初始命令或工作目录提示。
- 连接断开时保留标签壳和输出，重连后允许新建恢复会话。

### 交互层

- 选择即复制、右键粘贴、搜索输出、清屏、复制带提示符片段。
- 字体、字号、主题、光标、滚动行为、快捷键偏好。
- 命令执行中状态提示：运行中、退出码、断开。
- 长输出不阻塞窗口，支持快速滚动到底部和暂停自动跟随。

### 协作层

- 从文件管理器在当前目录打开终端。
- 从命令收藏夹把命令插入或直接运行。
- 向会话录制上报命令和可选输出摘要。
- 将选中文本发送到便签或复制为排障片段。

## 交互设计

- 顶部为标签栏和会话操作：新建、拆分、搜索、设置。
- 主区域为终端画布，拆分时每个 pane 有独立焦点和标题。
- 底部可选状态条展示编码、尺寸、连接状态和最后退出码。
- 非常用操作放入菜单，不在终端画布上堆按钮。

## 数据与状态

```ts
interface TerminalWorkspaceState {
  tabs: TerminalTabState[];
  activeTabId: string;
}

interface TerminalTabState {
  id: string;
  title: string;
  panes: TerminalPaneState[];
  activePaneId: string;
  disconnected: boolean;
}

interface TerminalPaneState {
  id: string;
  terminalSessionId: string;
  status: 'idle' | 'running' | 'exited' | 'disconnected';
  lastExitCode?: number;
}
```

## 能力与集成设计

- SSH shell 会话仍应由主进程维护，渲染层只消费终端流事件。
- 若支持初始命令和指定工作目录，需要扩展终端启动参数。
- 终端输出事件需要可选订阅策略，避免录制与 UI 都无限保存完整输出。
- 不把数据库、文件上传等高层动作塞进终端组件，只保留入口协作。

## 开发计划

1. 抽出终端 workspace 状态，支持多标签。
2. 增加 pane 拆分和焦点管理。
3. 增加搜索输出、暂停跟随、状态条。
4. 扩展新建会话参数，支持 title 和初始命令。
5. 接入文件管理器与命令收藏夹跳转。
6. 接入会话录制事件回调。
7. 补断线、重连、关闭多会话时的生命周期测试。

## 验收标准

- 同一连接可以稳定打开多个终端会话。
- 标签和分屏切换不会串写输入。
- 大量输出时 UI 仍可滚动和搜索。
- 连接关闭后终端显示明确状态，不悄悄丢内容。
- 与文件管理器和命令模板的跳转 payload 可用。

## 设计取舍

- 首版不做完整 shell 历史数据库，命令历史仍优先依赖远端 shell。
- 首版不做 tmux 替代品，分屏只服务 UI 并行查看。
- 终端仍是基础组件，应比扩展组件更稳定、更少依赖。
