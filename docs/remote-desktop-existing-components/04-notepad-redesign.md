# 记事本组件重设计文档

## 定位

记事本是远程文本文件编辑器，负责轻量文本、脚本和配置文件修改。它应比普通 textarea 更可靠，但不需要承担完整代码 IDE 的复杂度。

## 重新设计目标

- 支持多文件工作流和保存安全。
- 让远程文件冲突、编码、未保存状态清晰可见。
- 给脚本和配置文件足够的阅读体验：高亮、搜索、行号、差异预览。
- 和文件管理器、结构化配置编辑器、Diff 组件边界清楚。

## 功能架构

### 文件工作区

- 多标签文件。
- 新建未命名文本。
- 打开路径、最近文件、从文件管理器打开。
- 未保存标记和关闭确认。

### 编辑能力

- 语法高亮、行号、搜索替换。
- 自动换行、缩进显示、编码与换行符提示。
- 只读模式，适合权限不足或预览。
- 保存、另存为、保存前差异预览。

### 安全保存

- 打开时记录远程版本摘要。
- 保存前检测远程文件是否变化。
- 冲突时提供重新加载、覆盖、另存三种选择。
- 对二进制和超大文件给出拒绝或只读提示。

## 交互设计

- 顶部为文件标签和保存状态。
- 工具栏保留打开、保存、搜索、换行设置、语法模式。
- 主编辑区尽量沉浸，状态条显示路径、行列、编码、只读/修改状态。
- 弹窗一律自定义，不能依赖 sandbox 下不可用的系统 prompt/confirm。

## 数据与状态

```ts
interface NotepadTab {
  id: string;
  filePath?: string;
  title: string;
  content: string;
  originalContent: string;
  dirty: boolean;
  readOnly: boolean;
  revisionHint?: string;
}
```

## 能力与集成设计

- 文件读写继续使用 SFTP 相关 IPC。
- 差异预览可复用 Diff 工具函数。
- 结构化 JSON/YAML/TOML 可提供“用结构化编辑器打开”入口。
- 会话录制只记录保存动作摘要，不默认记录完整文件内容。

## 开发计划

1. 重整 tab 模型和未保存生命周期。
2. 增加搜索替换、行号、状态条。
3. 实现保存前远程变更检测。
4. 实现 diff 预览和冲突处理。
5. 增加最近文件与打开方式入口。
6. 细化文本/二进制/大文件策略。
7. 补跨系统路径、换行和保存失败测试。

## 验收标准

- 多标签编辑不会互相覆盖内容。
- 未保存关闭有确认。
- 远程文件外部变化能被检测到。
- 保存失败不丢当前编辑内容。
- 大文件和二进制文件策略清晰。

## 设计取舍

- 首版不做 LSP、调试、项目级搜索。
- 高级结构化编辑交给专门组件。
- 记事本必须优先保证保存安全和文本可靠性。
