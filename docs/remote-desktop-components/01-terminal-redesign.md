# 终端组件重设计文档

## 定位

终端是 ShellDesk 远程桌面的基础能力，不应只是一个 SSH 输出框，而应成为远程工作流的命令执行核心。重新设计时优先考虑长时间使用、可恢复上下文和与其他组件协作。并行排障继续通过远程桌面打开多个终端窗口承载，不在单个终端窗口内引入额外的会话布局复杂度。

## 重新设计目标

- 让单个终端窗口专注于一个稳定、清晰的 SSH shell 会话。
- 让执行长命令、查找历史输出、复制排障片段更顺手。
- 让终端能和文件管理器、命令收藏夹、部署面板、会话录制共享上下文。
- 保持终端本体轻、快、可靠，避免把它做成半个 IDE。

## 功能架构

### 会话层

- 单窗口单会话，窗口标题可展示当前会话 title 和连接状态。
- 新建终端窗口可选择默认 shell、初始命令或工作目录提示。
- 连接断开时保留终端壳和输出，重连后允许重新创建恢复会话。
- 关闭运行中的终端窗口时给出确认，避免误关长任务。

### 交互层

- 选择即复制、右键粘贴、搜索输出、清屏、复制带提示符片段。
- 字体、字号、主题、光标、滚动行为、快捷键偏好。
- 窗口标题栏展示终端会话状态提示：运行中、退出、断开。
- 长输出不阻塞窗口，支持快速滚动到底部和暂停自动跟随。

### 协作层

- 从文件管理器在当前目录打开终端。
- 从命令收藏夹把命令插入或直接运行。
- 向会话录制上报命令和可选输出摘要。
- 将选中文本发送到便签或复制为排障片段。

## 交互设计

- 标题栏提供三点终端菜单，承载新建终端窗口、搜索、清屏、自动跟随、滚动到底部和设置。
- 主区域为单一终端画布，优先保证输入焦点和输出阅读稳定。
- 终端画布内不保留额外工具栏和底部状态栏，连接状态统一收敛到窗口标题栏。
- 非常用操作放入菜单，不在终端画布上堆按钮。

## 数据与状态

```ts
interface TerminalSessionState {
  id: string;
  title: string;
  terminalSessionId: string;
  status: 'idle' | 'running' | 'exited' | 'disconnected';
  disconnected: boolean;
  lastExitCode?: number;
}
```

## 能力与集成设计

- SSH shell 会话仍应由主进程维护，渲染层只消费终端流事件。
- 若支持初始命令和指定工作目录，需要扩展终端启动参数。
- 终端输出事件需要可选订阅策略，避免录制与 UI 都无限保存完整输出。
- 不把数据库、文件上传等高层动作塞进终端组件，只保留入口协作。

## 开发计划

1. 抽出单终端会话状态，统一连接、断线、退出和关闭流程。
2. 增加搜索输出、暂停跟随和标题栏终端菜单。
3. 扩展新建会话参数，支持 title 和初始命令。
4. 接入文件管理器与命令收藏夹跳转。
5. 接入会话录制事件回调。
6. 补断线、重连、关闭终端窗口时的生命周期测试。

## 验收标准

- 同一连接可以稳定打开多个终端窗口，各窗口会话互不串写输入。
- 大量输出时 UI 仍可滚动和搜索。
- 连接关闭后终端显示明确状态，不悄悄丢内容。
- 与文件管理器和命令模板的跳转 payload 可用。

## 设计取舍

- 首版不做完整 shell 历史数据库，命令历史仍优先依赖远端 shell。
- 不做多标签终端和 pane 分屏，并行任务通过多个终端窗口承载。
- 不做 tmux 替代品，不在终端内承担复杂会话编排。
- 终端仍是基础组件，应比扩展组件更稳定、更少依赖。
