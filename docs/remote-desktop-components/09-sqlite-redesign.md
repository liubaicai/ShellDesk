# SQLite 管理器组件重设计文档

> 当前状态：已接入远程桌面（appKey: `sqlite`），实现入口为 `src/components/remote-desktop/RemoteSqlite.tsx`。本文保留重设计背景，维护时以当前实现、`RemoteDesktopShell.tsx` 注册表和 `_example.md` 清单为准。

## 定位

SQLite 管理器用于打开远程文件系统中的 SQLite 数据库文件，浏览结构、查询数据并做轻量编辑。它天然连接文件管理器和数据库工作台。

## 重新设计目标

- 让“从文件到数据库”的入口自然。
- 提供表、视图、索引、列信息和查询体验。
- 对远程文件数据库的锁、写入风险和工具依赖表达清楚。
- 与 MySQL/PostgreSQL 共享数据库结果体验。

## 功能架构

### 文件打开

- 从路径输入、文件选择器、文件管理器打开。
- 最近数据库文件。
- 打开前检查 sqlite 工具和文件可读性。

### 对象浏览

- 表、视图、索引、列。
- 表数据默认分页或 limit。
- schema SQL 查看。

### 查询

- SQL 编辑和结果表。
- 查询历史当前会话保留。
- 显示执行耗时、错误、影响行数。

### 编辑

- 首版可对明确 rowid/主键目标做单元格编辑。
- 写入前提示数据库文件会被修改。
- 数据库锁定或写失败时保留查询上下文。

## 交互设计

- 未打开时显示文件打开面板。
- 打开后左侧对象树，右侧查询和结果区。
- 顶部显示数据库文件路径，可跳转文件管理器。
- schema 详情和数据浏览通过 tabs 切换。

## 数据与状态

```ts
interface SqliteWorkspaceState {
  sqliteSessionId: string;
  filePath: string;
  selectedObject?: string;
  queryDraft: string;
}
```

## 能力与集成设计

- 远程 SQLite 操作可以继续由 Rust 后端持有会话并执行远程命令。
- 与文件管理器的打开方式 registry 紧密集成。
- 通用结果表格和 SQL 编辑行为可与 MySQL/PostgreSQL 统一。
- 文件保存/覆盖不由 SQLite 管理器处理，数据库写操作通过查询层完成。

## 开发计划

1. 重整数据库文件打开和错误状态。
2. 增加对象树到 views/indexes/schema。
3. 接入通用查询/结果工作区。
4. 增加表数据浏览限制和分页策略。
5. 强化写入提示和锁错误反馈。
6. 增加文件管理器、最近文件跳转。
7. 补 sqlite3 不存在、文件损坏、权限不足测试。

## 验收标准

- 能从远程文件打开 SQLite 数据库。
- 能浏览表和列并查询数据。
- schema 与文件路径信息清楚。
- 数据库锁或查询错误可见。
- 默认查询不会无限读取大表。

## 设计取舍

- 首版不做数据库文件修复工具。
- 文件数据库的导入导出和备份另行设计。
- 入口协作和查询安全比复杂编辑更优先。
