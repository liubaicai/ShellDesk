# 进程管理器组件重设计文档

> 当前状态：已接入远程桌面（appKey: `procmanager`），实现入口为 `src/components/remote-desktop/RemoteProcessManager.tsx`。本文保留重设计背景，维护时以当前实现、`RemoteDesktopShell.tsx` 注册表和 `_example.md` 清单为准。

## 定位

进程管理器用于查看远程主机正在运行的进程、资源占用、父子关系和可控信号。它应与系统监视器、端口管理器、服务管理器互相协作。

## 重新设计目标

- 用可扫描表格回答“谁在吃资源”。
- 支持进程搜索、排序、树视图和详情。
- 让结束进程、发送信号等危险动作受控。
- 为端口、服务、日志排障提供跳转路径。

## 功能架构

### 列表

- PID、PPID、用户、CPU、内存、命令、启动时间、状态。
- 排序、搜索、按用户筛选。
- 刷新和自动刷新开关。

### 详情

- 完整命令行、cwd、环境摘要可后续加入。
- 父子进程关系。
- 打开的端口/服务归属作为联动。

### 操作

- 发送 TERM、KILL、INT 等有限信号。
- Windows 结束任务。
- 复制 PID、命令行。

## 交互设计

- 顶部是刷新、搜索、自动刷新、排序入口。
- 主区域为高密度进程表。
- 右侧详情面板显示当前进程和操作。
- 发送信号按钮在详情区，必须确认并说明信号含义。

## 数据与状态

```ts
interface RemoteProcessEntry {
  pid: number;
  ppid?: number;
  user?: string;
  cpuPercent?: number;
  memoryPercent?: number;
  state?: string;
  command: string;
}
```

## 能力与集成设计

- Linux 可通过 `ps`/`top` 快照，Windows 用 PowerShell。
- 系统监视器可打开进程管理器并预设排序或筛选。
- 端口管理器可以按 PID 跳进进程详情。
- 服务管理器可以显示服务主进程跳转。

## 开发计划

1. 重整列表字段、排序和搜索。
2. 增加详情面板和树关系视图。
3. 明确自动刷新策略，避免操作中列表跳动。
4. 增强信号确认和结果反馈。
5. 增加从端口/监控/服务跳转 payload。
6. 补 Windows 字段差异。
7. 测试高进程数和 kill 失败场景。

## 验收标准

- Linux/Windows 能显示基本进程列表。
- 搜索、排序、刷新行为稳定。
- 发送信号前有确认。
- 进程退出或 PID 不存在时反馈清楚。
- 从监控或端口工具能定位进程。

## 设计取舍

- 不在进程管理器内做完整 profiler。
- 环境变量、文件句柄等敏感信息后置。
- 进程终止能力必须谨慎而明确。
