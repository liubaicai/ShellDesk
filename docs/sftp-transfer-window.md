# SFTP 文件传输窗口

ShellDesk 提供独立的双栏 SFTP 文件传输窗口。入口位于主机卡片的三点菜单和主机列表操作栏；它复用主机的认证信息，但不会先打开远程桌面。

## 界面结构

- 顶部会话栏显示主机、端口、用户、协议状态和连接安全状态。
- 左栏浏览本地文件系统，右栏浏览远端 SFTP 文件系统；两侧目录树支持按三角按钮懒加载展开/收起子目录，点击目录名称才进入该目录，并都有独立的新建文件/目录入口、路径历史、上级目录、刷新、搜索、排序、多选和右键菜单。
- 中间操作栏根据当前选择执行上传或下载；未选择条目时传输当前目录的全部内容，并提供递归目录对比和单向同步入口。
- 顶部“传输队列”按钮可显示或隐藏底部队列；隐藏时双栏文件区域自动填满释放出的空间，新任务入队时会自动恢复显示。队列支持 1–4 个并发任务、真实字节进度、文件完成数、速度、剩余时间、暂停、继续、取消、重试和清理已完成任务。
- 重命名、删除和属性修改使用自定义对话框；远端属性对话框支持八进制权限和递归修改。
- 深色和浅色主题均有完整样式，窄窗口会收起目录树以保证文件表格可用。

设计基准见 [SFTP 文件传输窗口概念图](design/sftp-transfer-window-concept.png)。

## 后端边界

远端操作由 `src-tauri/src/remote_fs/sftp.rs` 实现：

- 使用 `russh_client::connect_authenticated` 建立并认证 russh 会话；
- 打开 SSH session channel 并请求 `sftp` subsystem；
- 使用 `russh-sftp` 完成目录、元数据、增删改、权限和文件流操作；
- 使用 256 KiB 缓冲区流式上传/下载，不把整个文件读入内存；
- 复用连接级传输进度与取消机制。

此窗口不调用系统 `ssh`、`sftp`、`sshpass` 或 OpenSSH fallback，也不通过 `exec` 命令模拟普通 SFTP 操作。

## IPC

远端通道统一以 `connection:sftp-` 开头，包括目录读取、状态、新建、删除、重命名、权限、上传和下载。本地双栏操作通过 `files:list-local-directory`、`files:stat-local-path` 和相应的增删改通道进入现有 `local_fs.rs`。

新增或修改通道时必须同步：

1. Rust dispatcher；
2. `src/tauriBridge.ts`；
3. `src/vite-env.d.ts`；
4. React 调用与错误文案。

## 传输语义

- 目录上传与下载会递归创建目录并逐文件流式复制。
- 暂停运行中的任务会安全取消当前传输；继续时从该任务开头重新执行，避免在没有服务端原子续传保证时产生静默损坏。
- 上传或下载前会检查目标当前目录的同名项目，并用自定义弹窗让用户选择“覆盖 / 合并”“跳过已存在项”或“取消”：同名文件覆盖，同名文件夹合并并覆盖其中的同名文件；选择跳过时仍会递归遍历同名文件夹，只跳过目标中已经存在的子文件或类型冲突子目录，其他内容继续传输；文件与文件夹类型不一致时会先明确提示，确认覆盖后再删除旧目标。删除目录会递归处理。
- 目录对比会在单个 SFTP 会话中递归枚举两侧目录树，以相对路径、条目类型和文件大小统计全部差异；单向同步会把包含差异的顶层文件或目录加入传输队列，不自动删除目标侧独有文件。

## 验证

```bash
pnpm typecheck
pnpm check:contracts
cargo fmt --manifest-path src-tauri/Cargo.toml --check
cargo check --manifest-path src-tauri/Cargo.toml
```

如仓库根目录存在完整的 `.env` 测试 SSH 变量，可再运行 live SSH 验证；除主机、端口、用户和认证信息外，测试还需要 `SHELLDESK_TEST_SSH_KNOWN_HOSTS_PATH`。不要在日志或回复中输出测试密码；缺少凭据或 known_hosts 路径时应明确跳过，而不是构造测试主机。
